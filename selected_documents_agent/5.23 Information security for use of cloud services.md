# Selected Documents for ISO 27001 Control 5.23 Information security for use of cloud services

## 1. Supplier Security Risk Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170099594

This policy is the most critical document for Control 5.23 as it establishes comprehensive processes for acquisition, evaluation, and management of supplier/cloud services. It includes security requirements evaluation, risk assessment procedures, contract negotiation guidelines, supplier security self-assessment, periodic audits, and exit strategies - all key elements required by the control for managing cloud service provider relationships.

## 2. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This umbrella policy provides the foundational security framework that governs all information security practices, including cloud service usage. It establishes security objectives, information risk management principles, access management requirements, and customer data security provisions that must be applied when using cloud services to ensure confidentiality, integrity, and availability requirements are met.

## 3. Company Managed Service Agreements
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170108066

This standard directly addresses service level management and agreements with external providers, including underpinning contracts with third parties. It provides the framework for defining, monitoring, and managing service level objectives and qualitative objectives in cloud service agreements, which is essential for establishing clear responsibilities and performance expectations with cloud service providers.

## 4. Risk Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170100789

This policy establishes the methodology for identifying, analyzing, and managing risks at both operational and enterprise levels. It is essential for assessing and managing information security risks associated with cloud service usage, including the identification of residual risks that must be accepted by appropriate management when using cloud services.

## 5. Encryption and Cryptography Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170105815

This policy addresses critical security requirements for protecting data at rest and in transit, including when stored or processed in cloud environments. It establishes encryption standards and key management practices that must be implemented to meet confidentiality and integrity requirements when organization data is stored or processed by cloud service providers.

## 6. S4A Disaster Recovery Plan
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170101158

This document addresses business continuity and disaster recovery capabilities, which are critical considerations when defining backup requirements, service availability expectations, and exit strategies for cloud services. It supports the control's requirement for ensuring appropriate backup, recovery, and service availability provisions in cloud service agreements.

## 7. Security Incident Reporting Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104051

This procedure establishes the process for reporting and managing security incidents, which is essential for implementing the control's requirement (point h) regarding procedures for handling information security incidents that occur in relation to the use of cloud services. It ensures incidents involving cloud service providers are properly detected, reported, and managed.

## 8. Change Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104688

This policy provides the framework for managing changes to IT services and systems in a controlled manner. It is relevant for managing changes when using multiple cloud services, handling cloud service provider changes to infrastructure or service delivery, and ensuring proper authorization and documentation when transitioning to, modifying, or exiting from cloud services.
