# Selected Documents for ISO 27001 Control 5.21 - Managing information security in the ICT supply chain

## 1. Supplier Security Risk Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170099594

This policy is directly aligned with Control 5.21 as it comprehensively addresses supplier security risk management throughout the entire supply chain lifecycle. It covers defining security requirements for ICT acquisitions, assessing supplier security posture, implementing security self-assessments, requiring suppliers to propagate security requirements to sub-contractors, monitoring supplier compliance, and managing supplier off-boarding—all core elements of the control.

## 2. Vulnerability Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/214794244

This policy supports Control 5.21 by addressing the identification, evaluation, and treatment of vulnerabilities in both native products and third-party components throughout the supply chain. It specifically covers vulnerabilities in supporting systems and third-party applications, which aligns with the control's guidance on monitoring product and service compliance and ensuring ICT products function as expected without unwanted features.

## 3. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

As the umbrella security policy, this document establishes the overarching framework that supports Control 5.21 implementation, including requirements for secure SDLC, third-party module management, customer data security, and supply chain-related security controls. It provides the governance structure necessary for managing information security risks associated with ICT products and services throughout their lifecycle.

## 4. Endpoint Software Audit Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/1994686607

This procedure directly supports Control 5.21's guidance on implementing processes to ensure components from suppliers are genuine and unaltered, and monitoring for unauthorized or prohibited software. It establishes systematic auditing of software installed on company devices, maintaining allow/block lists, and ensuring compliance with security restrictions—key aspects of supply chain security management.

## 5. License and hardware purchase or renewal process
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520415

This procedure addresses Control 5.21's requirement to define information security requirements for ICT product or service acquisition. It establishes a formal approval process for purchasing licenses and hardware, including security and procurement team involvement for significant orders, ensuring appropriate evaluation and authorization before acquiring ICT products and services.

## 6. Change Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104688

This policy supports Control 5.21 by establishing formal change control processes for all ICT systems and third-party software upgrades managed by the organization. It addresses the control's guidance on monitoring and managing changes throughout the supply chain, including third-party provider changes, software upgrades, and ensuring security considerations in all changes affecting production environments.

## 7. S4A License Tracking and Update Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104620

This procedure supports Control 5.21's guidance on managing the ICT component lifecycle and availability. It establishes processes for tracking software licenses from suppliers, ensuring continuity of critical components, and managing the risks associated with license expiration—addressing the control's requirement to manage risks of components no longer being available due to supplier issues.

## 8. Procedure - Service transition to prod
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/SUP/pages/160433255

This procedure supports Control 5.21 by establishing comprehensive validation processes for ICT services before production deployment. It includes security reviews, monitoring setup, validation that delivered services comply with security requirements, verification of third-party integrations, and ensuring proper security controls are in place—aligning with the control's guidance on validating that delivered ICT products and services comply with security requirements.
