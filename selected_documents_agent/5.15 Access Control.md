# Selected Documents for ISO 27001 Control 5.15 Access Control

## 1. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This overarching policy establishes the framework for access management including authentication, authorization, role-based access control, least privilege principle, and separation of duties. It directly addresses multiple aspects of Control 5.15 including access control rules, privileged access restrictions, and formal authorization processes.

## 2. Onboarding Procedure / Procedura wdrożenia
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104063

This procedure details the process of granting initial access rights to new employees, including LDAP account creation, group assignments, system access provisioning, and authentication setup. It directly supports the formal authorization of access requests and access control rule implementation mentioned in Control 5.15.

## 3. Offboarding procedure / Procedura odejścia z firmy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104057

This procedure covers the systematic revocation of access rights when personnel leave the organization, including account deactivation, access removal, and transfer of responsibilities. It ensures that access control rules are maintained throughout the employee lifecycle and prevents unauthorized access.

## 4. Employee Offboarding
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520666

This detailed technical procedure provides comprehensive steps for removing access across all systems (LDAP, Google, Jira, AWS, VPN, licenses, alarm codes). It supports enforcement of access control rules and ensures that physical and logical access is properly terminated to prevent unauthorized access.

## 5. Information Classification Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170101881

This policy defines information sensitivity levels and handling requirements which form the basis for determining access control rules based on business and information security requirements. It directly supports the need-to-know principle and information security levels referenced in Control 5.15.

## 6. Gerrit - access request
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170099777

This procedure demonstrates formal authorization processes for access requests to development repositories, including approval workflows and access right management. It exemplifies the implementation of access control functions including access request, authorization, and administration as required by Control 5.15.

## 7. Confluence Pages Access Request Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/876347426

This procedure establishes formal processes for requesting and approving access to documentation systems, demonstrating segregation of access control functions and formal authorization requirements. It supports the management of access rights and access authorization processes outlined in Control 5.15.

## 8. Open VPN
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107980

This policy governs remote access to internal networks through VPN, implementing logical access controls with LDAP-based authentication and group-based authorization. It demonstrates access control rules for network services and restrictions based on user identity and group membership as required by Control 5.15.
