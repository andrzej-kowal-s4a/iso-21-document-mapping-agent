# Selected Documents for ISO 27001 Control 6.7 Remote working

## 1. Bring Your Own Device (BYOD) Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106633

This policy directly addresses remote working scenarios by establishing comprehensive rules for using personal devices for work purposes. It covers authentication requirements, device security controls, network access restrictions, and data protection measures specifically designed for personnel working outside organizational premises, aligning with the control's requirements for securing information accessed remotely.

## 2. Mobile Device Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106965

This policy governs the use of mobile devices (smartphones, tablets, laptops) by staff for work, which is directly relevant to remote working environments. It addresses device configuration requirements, authentication mechanisms, data handling rules, and security controls for both corporate and private devices accessing organizational data outside premises, supporting the control's guidance on remote access security.

## 3. Encryption and Cryptography Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170105815

This policy is essential for remote working as it mandates encryption for data at rest and in transit, including full disk encryption for all devices and VPN requirements for external connections. It directly supports the control's requirement to protect information accessed and processed remotely by ensuring cryptographic protection of sensitive data.

## 4. Open VPN
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107980

This policy establishes VPN as the exclusive method for remote access to internal networks from external locations, implementing strong authentication via LDAP. It directly addresses the control's guidance on communication security requirements and secure remote access mechanisms, which are fundamental to protecting information during remote working activities.

## 5. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This umbrella policy provides the overarching security framework that governs remote working, including access management, physical security, personnel security, and system security requirements. It establishes fundamental security principles that apply to remote working scenarios and references specific controls for authentication, authorization, and remote connections to organizational resources.

## 6. Network General Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104105

This policy addresses network security controls essential for remote working, including VPN requirements, remote access restrictions, network segmentation, and device security standards. It directly supports the control's guidance on communication security, threat protection from unauthorized access, and secure configuration of wireless network services used in remote working environments.

## 7. Information Technology Acceptable Use Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106001

This policy establishes acceptable use rules for IT resources during remote work, including data protection requirements, device usage guidelines, password management, and physical security measures. It complements the remote working control by defining user responsibilities and behavioral expectations when accessing organizational information from remote locations.

## 8. Security Incident Reporting Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104051

This procedure is critical for remote working environments as it establishes the process for reporting security incidents, including those involving mobile devices, lost equipment, or breaches occurring at remote locations. It directly addresses the control's guidance on information security event reporting requirements for remote working scenarios.
