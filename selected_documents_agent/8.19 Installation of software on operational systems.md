# Selected Documents for ISO 27001 Control 8.19 Installation of software on operational systems

## 1. Change Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104688

This policy is directly relevant to Control 8.19 as it establishes formal processes for making changes to IT systems, including software installation and updates. It defines change types (Standard, Normal, Emergency), covers authorization requirements, testing procedures, rollback strategies, and documentation requirements - all key elements mentioned in the control guidance for secure software installation management.

## 2. Procedure - Service transition to prod
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/SUP/pages/160433255

This procedure directly addresses software installation on production systems. It includes comprehensive checklists for service transitions covering software updates, testing requirements, configuration management, rollback strategies, and audit logging. It implements the control's guidance on performing updates only by trained administrators with proper authorization and ensuring extensive testing before production installation.

## 3. Endpoint Software Audit Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/1994686607

This procedure ensures compliance with software installation restrictions by conducting systematic audits of software on company devices. It implements monitoring and control of software installations, maintains approved/prohibited software lists, and enforces rules on what software users can install - directly supporting the control's requirement to define strict rules on software installation.

## 4. Vulnerability Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/214794244

This policy is essential for Control 8.19 as it addresses the identification and remediation of technical vulnerabilities through software patches and updates. It covers vulnerability assessment in operational systems, treatment strategies including patching, and aligns with the control's guidance on applying software patches to remove security vulnerabilities.

## 5. Server management procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107313

This procedure manages the lifecycle of operational systems including software installation on servers. It covers authorization processes, environment preparation, change management for server configurations, and configuration control - supporting the control's requirements for managing software installation on operational systems with appropriate authorization.

## 6. Procedure - how to handle upgrade tickets
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/SUP/pages/160435629

This procedure addresses systematic software upgrades across multiple operational systems. It ensures coordinated handling of software updates, proper review and approval processes, and implementation tracking - supporting the control's guidance on performing updates with management authorization and maintaining audit logs of software updates.

## 7. Procedure - Deployment Validation Test
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170105878

This procedure implements the control's requirement for extensive testing before software installation by standardizing deployment validation tests. It defines the scope of tests after deployment and documentation requirements, ensuring that software is only installed after successful testing as required by the control guidance.

## 8. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This overarching security policy establishes the framework for secure software systems development and deployment. It includes requirements for change control, separation of duties, secure SDLC, testing before production deployment, and formal deployment processes - providing the policy foundation for implementing Control 8.19's secure software installation requirements.
