# Selected Documents for ISO 27001 Control 7.4 Physical Security Monitoring

## 1. Alarm codes management
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520699

This procedure directly addresses the management of alarm system codes and user access to the physical security alarm console. It covers the creation, deletion, and configuration of alarm codes for employees, which is essential for controlling and monitoring unauthorized physical access to premises as required by the control.

## 2. Office Alarm Code
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107751

This procedure establishes the process for obtaining and using alarm codes for office access, including arming and disarming the alarm system. It directly supports physical security monitoring by defining how employees should interact with the intruder alarm system and includes critical security incident reporting requirements when alarms are not properly armed.

## 3. Procedura korzystnia z biura po godzinach pracy / After-Hours workplace activity procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104529

This procedure governs after-hours workplace activity and monitoring requirements. It addresses physical security concerns during unoccupied periods when continuous monitoring is most critical, including requirements for alarm system usage, supervision, and ensuring premises security outside standard business hours.

## 4. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This umbrella security policy includes a dedicated Physical Security section that establishes requirements for security perimeters, security barriers, access controls, visitor escorts, and protection of information processing facilities. It provides the high-level policy framework that supports the implementation of physical security monitoring controls.

## 5. Employee Offboarding
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520666

This procedure includes critical tasks for removing alarm access codes and physical access permissions when employees leave the organization. It ensures that physical security monitoring systems remain effective by preventing unauthorized access from former employees who no longer require access to monitored premises.

## 6. Security Incident Reporting Procedure / Procedura zgłaszania incydentów związanych z bezpieczeństwem
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104051

This procedure establishes the process for reporting security incidents, including physical security breaches detected through monitoring systems. It ensures that events detected by surveillance systems, alarms, or other monitoring mechanisms are properly reported and managed, which is essential for the detective aspect of physical security monitoring.
