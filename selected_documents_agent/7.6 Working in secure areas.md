# Selected Documents for ISO 27001 Control 7.6 Working in secure areas

## 1. Procedura korzystnia z biura po godzinach pracy / After-Hours workplace activity procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104529

This procedure directly addresses working in secure areas by defining security measures for after-hours work including supervision requirements (avoiding unsupervised work), access controls, and rules for securing vacant office space. It covers personnel awareness, visitor management, and emergency procedures visibility, which are key elements of the control.

## 2. Office Alarm Code
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107751

This procedure implements physical security controls for secure areas through alarm system management. It defines who can access secure areas after hours, how to properly arm/disarm security systems, and includes incident reporting procedures for security breaches, directly supporting the control's requirement for physically locking and periodically inspecting vacant secure areas.

## 3. Alarm codes management
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520699

This internal helpdesk procedure details the technical management of alarm codes for secure areas including user creation, deletion, and access rights configuration. It supports the control by ensuring proper access control mechanisms are in place for securing vacant areas and managing who has physical access to secure zones.

## 4. Mobile Device Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106965

This policy directly addresses the control's requirement for controlling the carrying and use of user endpoint devices in secure areas. It defines rules for using mobile devices with corporate data, requirements for device protection, restrictions on unattended devices, and loss/damage reporting procedures, which are essential for protecting information in secure areas.

## 5. Bring Your Own Device (BYOD) Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106633

This policy complements the Mobile Device Policy by governing the use of personal devices in work environments including secure areas. It addresses security requirements for devices that access company systems and data, device authentication, and physical security measures, supporting the control's guidance on appropriately controlling user endpoint devices in secure areas.
