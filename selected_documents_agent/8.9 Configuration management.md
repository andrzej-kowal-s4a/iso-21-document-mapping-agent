# Selected Documents for ISO 27001 Control 8.9 Configuration management

## 1. Server management procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107313

This procedure directly addresses configuration management for hardware (servers/VMs) including establishing configurations, documenting resource requirements, managing configuration changes through a formal approval workflow, and maintaining configuration records throughout the server lifecycle.

## 2. Change Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104688

This policy establishes the formal process for managing all changes to IT systems configurations, including planning, evaluation, review, approval, implementation, and documentation of changes - which is a core requirement of the configuration management control.

## 3. Procedure - Service transition to prod
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/SUP/pages/160433255

This procedure ensures proper configuration management during service transitions by verifying configurations are tested, documented, and monitored before production deployment. It includes configuration validation, monitoring setup, and configuration review requirements that support the control's objectives.

## 4. Endpoint Software Audit Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/1994686607

This procedure supports configuration management by systematically auditing software configurations on company devices, maintaining approved/blocked software lists, and ensuring endpoint configurations comply with security standards and policies.

## 5. Network General Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104105

This policy establishes requirements for network configuration management including device hardening, security configurations, configuration backups, periodic configuration audits, and monitoring to verify configuration settings - directly supporting secure network configuration requirements.

## 6. Vulnerability Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/214794244

This policy addresses configuration management from a security perspective by identifying configuration vulnerabilities, evaluating misconfigurations against standards, and establishing processes to remediate configuration-related security issues in systems and infrastructure.

## 7. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This umbrella policy establishes overarching requirements for secure configuration of systems including configuration hardening, changing default values, formal change control procedures, and monitoring configurations to detect deviations - providing the governance framework for configuration management.
