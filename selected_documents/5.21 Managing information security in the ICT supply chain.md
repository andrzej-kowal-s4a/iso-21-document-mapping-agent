# Selected Documents for ISO 27001 Control 5.21 Managing information security in the ICT supply chain

## 1. Supplier Security Risk Management Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170099594

This is the most directly relevant document as it comprehensively addresses ICT supply chain security requirements. The policy includes specific security requirements for supply chain security, requiring suppliers to ensure their subcontractors comply with the same security standards. It covers secure SDLC requirements, vulnerability management, security certifications, and supply chain security provisions. The document establishes processes for evaluating ICT products and services, including RFI procedures and security assessments that directly support managing information security risks in the ICT supply chain.

## 2. Purchasing Procedure for Standard Hardware
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/1633452033

This procedure is highly relevant as it establishes the procurement process for ICT hardware components. It covers hardware standards, procurement guidelines, and equipment lifecycle management. The procedure ensures that hardware is procured from appropriate sources and follows standardized processes, which aligns with control 5.21 requirements for defining information security requirements for ICT product acquisition and ensuring components are genuine and unaltered. It addresses the need to obtain ICT from reputable sources and manage hardware lifecycle.

## 3. License and hardware purchase or renewal process
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4AIH/pages/149520415

This procedure is relevant because control 5.21 requires processes for managing ICT component lifecycle and availability, including managing risks of components no longer being available. This document establishes the procurement workflow for licenses and hardware, including approval processes, vendor selection, and renewal procedures. It ensures proper oversight of ICT acquisitions and helps identify alternative suppliers when needed, addressing the control requirement for managing ICT component lifecycle and associated security risks.

## 4. Endpoint Software Audit Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/1994686607

This procedure is relevant because control 5.21 requires implementing monitoring processes and methods for validating that delivered ICT products comply with stated security requirements. The document establishes procedures for auditing software installed on company devices, reviewing software against approved lists, and ensuring compliance with security restrictions. It supports the control requirement for monitoring and validating ICT products and identifying software components, which is essential for managing ICT supply chain security.

## 5. Server management procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107313

This procedure is relevant because control 5.21 addresses ICT infrastructure components and requires processes for managing ICT products and services. The document establishes procedures for requesting, provisioning, and managing virtual machines and server infrastructure. It ensures proper configuration management and tracking of ICT infrastructure components, supporting the control requirement for identifying and documenting product or service components that are critical for maintaining functionality and require increased attention in the supply chain.
