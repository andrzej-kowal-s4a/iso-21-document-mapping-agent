# Selected Documents for ISO 27001 Control 5.15 Access Control

## 1. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This policy is highly relevant as it contains a dedicated "Access Management" section that directly addresses ISO 27001 control 5.15 requirements. It establishes that access to IT systems shall be provided through individually assigned unique identifiers (User ID) with authentication tokens, mandates stronger authentication methods where appropriate, requires a process for application and approval of privileges, implements least privilege principle, specifies role-based authorization, and addresses privileged access restrictions. The policy also covers physical access controls and establishes the overall framework for access control rules.

## 2. Rules of Personal Data Protection
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170102152

This document is highly relevant as it establishes detailed procedures for access management including authorization procedures for work commencement, suspension and termination. It mandates that each user must have their own individual ID for authentication and authorization, specifies that accounts are managed by IT Administrators with authorizations granted based on business role, prohibits work on another user's account, and restricts use of common accounts. The document also covers password policies and secure workstation practices that support access control implementation.

## 3. OpenVPN Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170107980

This policy is relevant as it establishes access control rules for remote connections to internal networks, requiring exclusive use of OpenVPN for external access. It specifies authentication requirements based on LDAP accounts and group membership, ensuring that only authorized personnel can access internal resources. The policy supports the control's requirement for considering all types of available connections in distributed environments and ensuring entities are only provided with access to information and assets they are authorized to use.

## 4. Information Technology Acceptable Use Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106001

This policy is relevant as it establishes rules for user identity and password protection, requiring users to protect their security credentials appropriately and never use another person's credentials. It mandates logging out of systems after work and locking screens, and prohibits unauthorized access to information or systems unless there is a valid business need. The policy supports the control's requirements for access control rules and user responsibilities in maintaining access security.
