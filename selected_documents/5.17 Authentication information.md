# Selected Documents for ISO 27001 Control 5.17 Authentication information

## 1. Information Security Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170104722

This umbrella policy establishes the foundational requirements for access management and authentication, including provisions for unique identifiers, password protection, multi-factor authentication, and least privilege principles. It directly addresses authentication token management and the requirement to treat authentication information as secret, which are core requirements of Control 5.17. The policy mandates that authentication tokens be protected appropriately and encourages stronger authentication methods where feasible.

## 2. Product Development and Support Security Credentials Procedure
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106199

This procedure specifically addresses the handling of security credentials (passwords) throughout the software lifecycle, including secure transmission, prohibition of storing credentials in version control, secure channel communication, and changing default credentials. It directly implements the guidance requirements for allocation and management of authentication information outlined in Control 5.17, ensuring that credentials are transmitted securely and default credentials are changed immediately after installation.

## 3. Rules of Personal Data Protection
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170102152

This document contains a dedicated password policy section that establishes user responsibilities for password management, including requirements for strong passwords, confidentiality of passwords, prohibition of password sharing, and immediate password changes upon compromise. It directly addresses the user responsibilities section of Control 5.17 regarding appropriate handling of authentication information and provides specific guidance on password selection criteria.

## 4. Encryption and Cryptography Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170105815

This policy covers cryptographic key management, password storage and transmission in protected form, rotation requirements, and encryption standards. It directly supports Control 5.17's requirements for protecting authentication information through approved cryptographic techniques and secure storage/transmission of passwords. The policy ensures that passwords and authentication information are stored and transmitted in protected form as required by the control.

## 5. Information Technology Acceptable Use Policy
**Confluence URL:** https://smart4aviation.atlassian.net/wiki/spaces/S4A/pages/170106001

This policy establishes user responsibilities for protecting security credentials, including password protection, prohibition of credential sharing, logging out requirements, and phishing awareness. These provisions directly address the user responsibilities section of Control 5.17 regarding appropriate handling of authentication information, ensuring users understand their obligations to keep authentication information confidential and report suspicious activities.
